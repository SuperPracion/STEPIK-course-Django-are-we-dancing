Правильная установка Python
- Обязательно выбрать пункт "Add path python" в главном окне при запуске установщика


Правильная установка Django
- Запустить cmd
- Проверить корректность установленного python командой python --version (должна появиться версия)
- Вторая проверка корректности установка python командой pip (должна появиться простынка информации "команда - пояснение")
- Установка Django командой python install django (для конкретной версии Django python install django==<version>)
- Удаление  Django командой python uninstall django


Работа с командной строкой Windows
- dir используется для отображения файлов в текущей директории
- cd <путь/каталог> используется для смены текущей директории на указанный <путь/каталог>
- mkdir используется для создания каталога в текущей директории


Создание скелета программы через CMD
- Открыть cmd
- Создать папку в которой будут храниться все проекты на django
- Прописать команду django-admin startproject <project name>


Настройка GIT + Django
- Создать проект на Git'е
- Подключиться через PyCharm к созданному git проекту
- В терминале pycharm создать проект командой python django-admin startproject / Или закинуть в клонированную папку ранее созданный проект
- В терминале PyCharm запустить сервер командой python manage.py runserver
- Выключить сервер - сочетание клавиш ctrl + c


Что такое проект и что такое приложение в django?
Проект - это результирующий вид готового продукта. Это может быть сайт, одиночная страница и др.
Приложение - это маленькая часть проекта отвечающая за определённое поведение/логику. Это может быть реализация профиля, оплаты, описание товара и др.


Описание файлов в django приложении
- admin.py необходим для работы в админкой
- apps.py
- tests.py необходим для написания тестов к приложению
- views.py необходим для описания всей логики в созданном приложении (пакете)


Создание приложения в проекте
